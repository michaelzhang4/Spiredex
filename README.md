# Spiredex

**▶ Live: [michaelzhang4.github.io/Spiredex](https://michaelzhang4.github.io/Spiredex/)**

An interactive, **single-file** HTML reference for *Slay the Spire 2*. See each enemy's
turn‑by‑turn moves and attack patterns, which enemies appear together in every act's
encounters, plus events, ancients, and game mechanics — all cross‑linked so you don't
have to dig through separate spreadsheet tabs mid‑run.

> ⚠️ Unofficial fan reference. *Slay the Spire* and *Slay the Spire 2* are © [Mega Crit](https://www.megacrit.com/).
> This project is not affiliated with, endorsed by, or sponsored by Mega Crit.

## Features

- **Acts & Encounters** — browse each act's easy/hard combat pools, elites, and bosses;
  every encounter expands inline to show the full stat block (HP, moves, attack pattern)
  of each enemy in that fight, so you can see exactly what they'll do.
- **Bestiary** — all enemies (monsters, elites, bosses) in one searchable list, each
  showing which act encounters it appears in.
- **Events**, **Ancients**, and **Mechanics** reference sections.
- **Search** across everything, an **Ascension** toggle that highlights the upgraded
  values, and colour‑coded keywords (attack / block / buff / debuff / summon / status).
- **No install, works offline** — a single HTML file with no dependencies. Just open it.

## Usage

### View it
**Live site:** <https://michaelzhang4.github.io/Spiredex/>

Or download **`index.html`** and open it in any modern browser — it is fully
self‑contained and works offline.

### Update the data
The current CSVs are baked into the HTML as the default. When the source data changes,
load the new CSVs in the page (they're remembered in your browser until you press
**Reset**):

- **📁 Load folder** — pick the folder of CSVs; loads them all at once
- **↓ Files** — pick individual CSV files
- **drag‑and‑drop** CSV files anywhere on the page

Files are matched by name (e.g. `… - Monsters.csv`, `… - Bosses.csv`).

## Repository layout

| Path | What it is |
| --- | --- |
| `index.html` | The built, self‑contained app — what you open / what GitHub Pages serves |
| `data/Slay the Spire 2 Reference - *.csv` | Source data, one file per spreadsheet tab |
| `build/lib.js` | Pure parsing / name‑matching / rendering logic (no DOM; Node‑testable) |
| `build/app.js` | Browser UI bootstrap (views, search, CSV loading) |
| `build/template.html` | HTML shell + CSS, with placeholders for the data and scripts |
| `build/build.ps1` | Assembles the final HTML from the template + scripts + CSVs |
| `build/test.js`, `build/verify.js` | Validation harnesses (Node) |

## Building from source

The HTML is generated by inlining the CSVs and the two scripts into the template.

```powershell
# from the repo root (Windows PowerShell or PowerShell 7)
powershell -ExecutionPolicy Bypass -File "build/build.ps1"
```

Validate (requires [Node.js](https://nodejs.org/)):

```bash
node build/test.js     # parses the source CSVs and reports name‑match coverage
node build/verify.js   # re-extracts the embedded data from the built HTML and checks it
```
